# Security Policy

## Supported Versions

This repository is an early sample app. Security fixes target the `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for sensitive reports. Use GitHub private
vulnerability reporting if it is enabled for the repository, or contact a
maintainer privately.

Include:

- Affected commit or version.
- Reproduction steps.
- Expected impact.
- Any relevant browser, OpenCode, Node.js, or package versions.

## Scope

This project is a browser UI for a local agent harness. Reports are most useful
when they involve unsafe rendering, prompt/tool approval UI bugs, dependency
vulnerabilities with a reachable path, or accidental exposure of local runtime
configuration.
